# Inspira — ChatGPT Custom GPT Actions

This guide sets up a ChatGPT Custom GPT that can read and write an
Inspira canvas on your behalf, using your existing ChatGPT Plus or Pro
subscription. OpenAI hosts the model; Inspira hosts the canvas. No LLM
tokens are billed to your Inspira account — we simply persist whatever
the model on the other end says you said.

The integration uses **OpenAI's Custom GPT Actions** feature (not MCP).
ChatGPT Actions speak plain OpenAPI over HTTPS; Inspira autogenerates
the schema via FastAPI. The same 11 tool operations are exposed to
Claude via MCP — see `docs/integrations/claude-mcp-connector.md` for
that path.

## Prerequisites

1. A ChatGPT Plus or Pro subscription (required to build Custom GPTs).
2. An Inspira account at https://tryinspira.com.
3. A Personal Access Token minted from
   **Account Settings → API tokens**. You'll paste it into the ChatGPT
   builder in step 4.

## 1. Create the Custom GPT

1. In ChatGPT, open **Explore GPTs → Create**.
2. In the **Configure** tab, give the GPT a name (e.g. "Inspira") and
   a short description:

   > Plan ideas on your Inspira canvas. I'll ask questions, you answer,
   > and every exchange saves automatically to the right topic.

3. Paste the following **Instructions** into the system-prompt box:

   ```
   You are an interviewer attached to the user's Inspira canvas.
   Inspira is a thinking space, not a chat window — every meaningful
   exchange must persist to a topic on the canvas so the user can
   revisit and rearrange later.

   When the user gives you an idea, call `create_canvas` and add
   topics with `add_topic` — one per distinct angle. As the user
   answers, call `record_answer(topic_id, question, answer)` so the
   turn lands on the right topic. Turn the user's explicit commitments
   into decisions via `add_decision`.

   Prefer short questions (≤ 2 sentences). Never duplicate what's
   already on the canvas — call `list_topics` before adding. When the
   user asks "what have we got?", call `get_summary` and paraphrase
   briefly.
   ```

## 2. Add the Actions schema

1. Scroll to **Actions** in the Configure tab and click **Create new
   action**.
2. Under **Schema**, paste the URL of Inspira's autogenerated OpenAPI
   document:

   ```
   https://api.tryinspira.com/openapi.json
   ```

   ChatGPT will pull the schema and filter down to the `mcp` tag group
   — that's the 11 operations exposed to the MCP + Actions surface.

3. Under **Authentication**, pick **API Key**.
   - **API Key**: paste the `inspira_pat_...` token you minted in
     Inspira's Account Settings.
   - **Auth Type**: `Bearer`.
   - **Custom Header Name**: leave empty (Bearer auth uses
     `Authorization`).

4. Save the action. ChatGPT will run a quick "test available actions"
   probe — if everything is wired correctly, you'll see the 11 tool
   names: `create_canvas`, `list_projects`, `list_topics`, `add_topic`,
   `update_topic`, `delete_topic`, `add_relationship`, `record_answer`,
   `add_decision`, `get_summary`, `export_markdown`.

## 3. Add starter prompts

In the **Conversation starters** list, paste the four suggestions the
launch memo calls out:

- Plan a week-long solo writing retreat.
- Help me think through switching careers to design.
- Draft an outline for a 20-minute talk on deep work.
- What's on my Inspira canvas? Let's pick up where we left off.

## 4. Test it

In the preview pane on the right, say:

> Plan a week-long solo writing retreat.

The GPT should call `create_canvas`, get a `project_id` back, then
follow up with 4–6 `add_topic` calls and ask its first question. Watch
the tool-call panel — if a call fails with a 401, re-check that the
PAT hasn't been revoked; if it fails with a 404, you're probably
referring to a topic id from a different canvas.

## 5. Publish

1. Click **Save → Only me** (or **Anyone with a link**; do NOT
   publish broadly until you're confident in the UX).
2. Copy the share URL.

## 6. Rotate the PAT

PATs are long-lived. Revoke any token you're not using via
**Account Settings → API tokens → Revoke**. To rotate:

1. Mint a new token in Inspira.
2. Edit the Custom GPT, replace the Authentication → API Key value,
   save.
3. Revoke the old token.

## Troubleshooting

- **401 on every call** — the PAT is wrong or has been revoked. Mint
  a fresh one and update the Custom GPT.
- **429 on bursts** — Inspira rate-limits per PAT; wait a few seconds
  and retry. If this happens routinely, open an issue.
- **"Actions" not visible** — your ChatGPT plan doesn't include GPT
  builder features. Upgrade to Plus or Pro.
- **Tools missing from the list** — the OpenAPI schema filter dropped
  something unexpectedly. Re-paste the schema URL; ChatGPT caches
  occasionally.
